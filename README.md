# Mini-GPT

从零手搓一个小型 GPT 模型，目的不是做产品，是搞懂它。

## 这个项目是什么

用 Python + PyTorch 从零搭建一个 ~10M 参数的 GPT 模型，在本地 Mac CPU 上训练，看它从随机输出变成能生成文字。

## 为什么做这个

读完了 Nested Learning 论文后，想亲手摸一摸模型到底是怎么工作的。

**学习目标：**
- Transformer 到底是怎么工作的？
- 训练过程发生了什么？
- 一个模型是怎么从"随机噪声"变成"能说话"的？

## 这个项目不是什么

- 不是要做一个能用的 AI 产品
- 不追求性能指标或 benchmark
- 不和任何模型比高低

## 核心原则

**写代码 → 讲原理 → 理解了 → 下一步。**

Claude Code 写代码很快，但每写完一个模块都要停下来，搞懂它在干什么，然后再继续。不是一口气写完对着一堆代码发呆。

## 技术栈

- **Python**：深度学习领域的标准语言
- **PyTorch**：主流深度学习框架，负责矩阵运算和自动求导

## 学习路线

### 第一阶段：用 PyTorch 搭建完整 GPT

| 步骤 | 做什么 | 学到什么 |
|------|--------|----------|
| 1 | 字符编码（Tokenizer） | 机器怎么"看懂"文字 |
| 2 | 搭模型骨架 | Embedding、Attention、FFN 是什么 |
| 3 | 写训练循环 | 模型怎么从错误中学习 |
| 4 | 跑第一次训练 | 看它从乱码变成像样的文字 |
| 5 | 生成文本 | 模型怎么一个字一个字地"说话" |
| 6 | 实验和玩 | 改参数看效果，建立直觉 |

### 第二阶段：深入底层，选择性手写核心组件

不替换整个 PyTorch（那是造计算器，不是学数学），而是手写最有启发性的部分：

| 手写什么 | 理解什么 |
|---------|---------|
| 自动求导引擎（micrograd） | 模型到底怎么"学习"的，梯度怎么一层层传回来 |
| 纯数值实现 Attention | Q、K、V 不再是抽象符号，看到每一步数字的变化 |
